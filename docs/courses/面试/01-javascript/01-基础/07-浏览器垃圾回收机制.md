## 内存管理基础

### 栈内存与堆内存

#### 栈内存（Stack）

- 存储基本数据类型（Number、String、Boolean、Undefined、Null、Symbol、BigInt）
- 存储引用类型的内存地址
- 自动分配和释放，遵循 LIFO（后进先出）原则
- 大小固定，存取速度快
- 由操作系统自动管理

```javascript
function example() {
  let num = 10; // 栈内存
  let str = "hello"; // 栈内存
  let obj = { a: 1 }; // 栈内存存储引用地址，堆内存存储对象
}
```

#### 堆内存（Heap）

- 存储引用数据类型（Object、Array、Function 等）
- 动态分配，大小不固定
- 存取速度相对较慢
- 需要垃圾回收机制管理

```javascript
const obj1 = { name: "Alice" }; // 堆内存
const obj2 = obj1; // 栈内存复制引用地址
obj2.name = "Bob"; // 修改堆内存中的对象
console.log(obj1.name); // "Bob"
```

### 内存生命周期

- **分配内存**：声明变量、函数、对象时分配内存
- **使用内存**：读写操作
- **释放内存**：垃圾回收器自动回收不再使用的内存

## 垃圾回收算法

### 引用计数（Reference Counting）

**原理**：跟踪每个值被引用的次数，当引用次数为 0 时回收。

**优点**：

- 实现简单
- 回收及时

**缺点**：

- 无法解决循环引用问题
- 需要额外空间存储引用计数

```javascript
// 循环引用问题
function circularReference() {
  let obj1 = {};
  let obj2 = {};
  obj1.ref = obj2;
  obj2.ref = obj1;
  // 函数执行完毕后，obj1 和 obj2 互相引用，引用计数永远不为 0
}
```

### 标记清除（Mark-Sweep）

**原理**：

- 标记阶段：从根对象（全局对象、活动函数的局部变量）开始遍历，标记所有可达对象
- 清除阶段：清除未被标记的对象

**优点**：

- 解决循环引用问题
- 主流浏览器采用的基础算法

**缺点**：

- 产生内存碎片
- 清除阶段会造成程序暂停（Stop-The-World）

```javascript
// 标记清除可以正确处理循环引用
function example() {
  let obj1 = {};
  let obj2 = {};
  obj1.ref = obj2;
  obj2.ref = obj1;
  // 函数执行完毕后，obj1 和 obj2 不可达，会被回收
}
```

### 标记整理（Mark-Compact）

在标记清除的基础上，增加整理阶段，将存活对象移动到一端，清理边界外的内存，解决内存碎片问题。

### 分代回收（Generational Collection）

**核心思想**：根据对象存活时间将内存分为新生代和老生代，采用不同的回收策略。

## V8 引擎的垃圾回收机制

### 内存结构

V8 将堆内存分为：

- **新生代（Young Generation）**：存储生命周期短的对象，容量小（1-8MB）
- **老生代（Old Generation）**：存储生命周期长的对象，容量大

### 新生代回收：Scavenge 算法

**原理**：采用 Cheney 算法，将新生代分为两个半区（Semispace）

- **From 空间**：使用中的内存区域
- **To 空间**：空闲区域

**回收过程**：

- 检查 From 空间的对象
- 将存活对象复制到 To 空间
- 清空 From 空间
- 交换 From 和 To 空间

**晋升策略**（对象从新生代移到老生代）：

- 对象经历过一次 Scavenge 回收
- To 空间使用超过 25%

```javascript
// 新生代对象示例
function createTemporaryObjects() {
  for (let i = 0; i < 1000; i++) {
    let temp = { data: new Array(100) }; // 短生命周期对象
  }
}
```

### 老生代回收：Mark-Sweep & Mark-Compact

**标记清除**：主要使用标记清除算法

**标记整理**：当内存碎片过多时触发

**增量标记（Incremental Marking）**：

- 将标记过程分成多个小步骤
- 与 JavaScript 执行交替进行
- 减少停顿时间

**并发标记（Concurrent Marking）**：

- 标记过程在后台线程执行
- 不阻塞主线程

**并行回收（Parallel Scavenge）**：

- 使用多个辅助线程同时执行回收
- 缩短回收时间

**惰性清理（Lazy Sweeping）**：

- 清除过程按需进行
- 不需要一次性清除所有垃圾

### 优化技术

**写屏障（Write Barrier）**：

- 记录老生代对象对新生代对象的引用
- 避免全堆扫描

**黑白灰三色标记**：

- 白色：未被标记的对象
- 灰色：已标记但其引用未被扫描的对象
- 黑色：已标记且其引用已被扫描的对象

## 内存泄漏

### 常见内存泄漏场景

**意外的全局变量**

```javascript
function leak() {
  name = "global variable"; // 未使用 let/const/var
  this.age = 18; // 非严格模式下 this 指向 window
}
```

**未清理的定时器和回调**

```javascript
let data = {
  /* 大量数据 */
};

setInterval(() => {
  console.log(data);
}, 1000);

// 应该在不需要时清理
const timer = setInterval(() => {
  console.log(data);
}, 1000);

// 清理定时器
clearInterval(timer);
```

**DOM 引用**

```javascript
const elements = {
  button: document.getElementById("button"),
};

// DOM 元素被删除，但 elements 仍持有引用
document.body.removeChild(document.getElementById("button"));
// elements.button 仍占用内存
```

**闭包**

```javascript
function outer() {
  const largeData = new Array(10000).fill("data");

  return function inner() {
    console.log("inner function");
    // inner 函数保持对 outer 作用域的引用
    // 即使不使用 largeData，它也不会被回收
  };
}

const fn = outer();
```

**正确做法**：

```javascript
function outer() {
  const largeData = new Array(10000).fill("data");

  return function inner() {
    console.log(largeData[0]); // 只引用需要的数据
  };
}
```

**事件监听器未移除**

```javascript
const element = document.getElementById("button");

function onClick() {
  console.log("clicked");
}

element.addEventListener("click", onClick);

// 应该在不需要时移除
element.removeEventListener("click", onClick);
```

**WeakMap/WeakSet 的误用**

```javascript
// 普通 Map 会阻止垃圾回收
const map = new Map();
let obj = { data: "value" };
map.set(obj, "metadata");
obj = null; // obj 仍无法被回收

// WeakMap 允许垃圾回收
const weakMap = new WeakMap();
let obj2 = { data: "value" };
weakMap.set(obj2, "metadata");
obj2 = null; // obj2 可以被回收
```

### 内存泄漏检测

**Chrome DevTools Memory Profiler**

- Heap Snapshot：拍摄堆快照，比较不同时间点的内存
- Allocation Timeline：记录内存分配时间线

实时监控内存使用情况

## 内存优化最佳实践

### 编码规范

```javascript
// 1. 及时解除引用
let largeObject = {
  /* 大量数据 */
};
// 使用完毕后
largeObject = null;

// 2. 使用局部变量
function process() {
  const data = fetchData(); // 局部变量，函数执行完自动回收
  return data.result;
}

// 3. 避免不必要的闭包
// ❌ 不好
function createHandler() {
  const largeData = new Array(10000);
  return () => console.log("handler"); // 保留了 largeData 引用
}

// ✅ 好
function createHandler() {
  return () => console.log("handler");
}

// 4. 使用事件委托
// ❌ 不好：为每个元素添加监听器
document.querySelectorAll(".item").forEach((item) => {
  item.addEventListener("click", handleClick);
});

// ✅ 好：使用事件委托
document.querySelector(".container").addEventListener("click", (e) => {
  if (e.target.classList.contains("item")) {
    handleClick(e);
  }
});
```

### 数据结构选择

```javascript
// 1. 大量数据使用 TypedArray
const buffer = new ArrayBuffer(1000000);
const int32Array = new Int32Array(buffer);

// 2. 弱引用集合
const cache = new WeakMap();
const listeners = new WeakSet();

// 3. 使用对象池
class ObjectPool {
  constructor(createFn) {
    this.pool = [];
    this.createFn = createFn;
  }

  acquire() {
    return this.pool.pop() || this.createFn();

  release(obj) {
    this.pool.push(obj);
  }
```

## 面试高频问题

### Q：JavaScript 的内存管理是自动的吗？

是的，JavaScript 具有自动内存管理机制（垃圾回收）。开发者不需要手动分配和释放内存，但需要注意避免内存泄漏。

### Q：V8 为什么要分代回收？

基于**分代假说**：

- 大部分对象生命周期很短
- 不死的对象会活得更久

分代回收针对不同生命周期的对象使用不同策略，提高回收效率。新生代使用空间换时间的 Scavenge 算法，老生代使用标记清除+标记整理。

### Q：如何判断对象是否可回收？

从 GC Roots（全局对象、活动函数栈、活动的闭包）开始，通过引用链追踪可达对象。不可达的对象即为垃圾，可以被回收。

### Q：什么是 Stop-The-World？如何优化？

Stop-The-World 是指垃圾回收时暂停 JavaScript 执行。V8 的优化方案：

- **增量标记**：分步执行，减少单次暂停时间
- **并发标记**：后台线程执行，不阻塞主线程
- **并行回收**：多线程同时回收，缩短总时间
- **惰性清理**：按需清理，分散压力

### Q：WeakMap 和 Map 的区别？

- **Map**：强引用，键值不会被垃圾回收
- **WeakMap**：弱引用，键可以被垃圾回收，键只能是对象，不可遍历

使用场景：WeakMap 适合存储对象的元数据，避免内存泄漏。

### Q：如何避免内存泄漏？

- 避免意外的全局变量
- 及时清理定时器和事件监听器
- 避免不必要的闭包引用
- 注意 DOM 引用
- 使用 WeakMap/WeakSet 处理对象关联数据
- 合理使用缓存，设置缓存上限

### Q：闭包一定会造成内存泄漏吗？

不一定。闭包只是保持了对外部作用域的引用，只有当闭包不再被使用但仍被引用时才会造成内存泄漏。合理使用闭包并及时解除引用不会造成泄漏。

## 实战案例

### 实现一个内存安全的缓存

```javascript
class LRUCache {
  constructor(capacity) {
    this.capacity = capacity;
    this.cache = new Map();
  }

  get(key) {
    if (!this.cache.has(key)) return -1;

    const value = this.cache.get(key);
    // 更新访问顺序
    this.cache.delete(key);
    this.cache.set(key, value);
    return value;
  }

  put(key, value) {
    if (this.cache.has(key)) {
      this.cache.delete(key);
    }

    this.cache.set(key, value);

    // 超出容量，删除最久未使用的
    if (this.cache.size > this.capacity) {
      const firstKey = this.cache.keys().next().value;
      this.cache.delete(firstKey);
    }
  }
}
```

### 实现带过期时间的缓存

```javascript
class TimedCache {
  constructor() {
    this.cache = new Map();
    this.timers = new Map();
  }

  set(key, value, ttl) {
    // 清除旧定时器
    if (this.timers.has(key)) {
      clearTimeout(this.timers.get(key));
    }

    this.cache.set(key, value);

    const timer = setTimeout(() => {
      this.cache.delete(key);
      this.timers.delete(key);
    }, ttl);

    this.timers.set(key, timer);
  }

  get(key) {
    return this.cache.get(key);
  }

  clear() {
    // 清理所有定时器，避免内存泄漏
    for (const timer of this.timers.values()) {
      clearTimeout(timer);
    }
    this.cache.clear();
    this.timers.clear();
  }
}
```

### 内存泄漏检测工具

```javascript
class MemoryLeakDetector {
  constructor() {
    this.snapshots = [];
  }

  takeSnapshot() {
    if (performance.memory) {
      this.snapshots.push({
        timestamp: Date.now(),
        usedJSHeapSize: performance.memory.usedJSHeapSize,
        totalJSHeapSize: performance.memory.totalJSHeapSize,
      });
    }
  }

  analyze() {
    if (this.snapshots.length < 2) return null;

    const first = this.snapshots[0];
    const last = this.snapshots[this.snapshots.length - 1];

    const growth = last.usedJSHeapSize - first.usedJSHeapSize;
    const growthRate = (growth / first.usedJSHeapSize) * 100;

    return {
      initialMemory: first.usedJSHeapSize,
      currentMemory: last.usedJSHeapSize,
      growth,
      growthRate: growthRate.toFixed(2) + "%",
      suspicious: growthRate > 50, // 增长超过 50% 视为可疑
    };
  }
}

// 使用示例
const detector = new MemoryLeakDetector();
detector.takeSnapshot();

// 执行一些操作
for (let i = 0; i < 10000; i++) {
  // 模拟操作
}

detector.takeSnapshot();
console.log(detector.analyze());
```

## 总结

- JavaScript 的垃圾回收主要采用**标记清除**算法，解决了循环引用问题
- V8 使用**分代回收**策略，针对新生代和老生代采用不同算法
- 现代引擎通过**增量标记**、**并发标记**、**并行回收**等技术优化性能
- 开发者需要注意**内存泄漏**问题：全局变量、定时器、闭包、DOM 引用等
- 使用 Chrome DevTools 进行内存分析和泄漏检测
- 遵循最佳实践：及时解除引用、使用弱引用集合、事件委托、对象池等

理解垃圾回收机制有助于编写高性能、内存安全的 JavaScript 应用。
