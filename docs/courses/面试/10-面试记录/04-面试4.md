## AI相关

### claude code 最佳实践

**用 CLAUDE.md 写清楚项目上下文**

CLAUDE.md 是 Claude Code 每次启动都会读取的指令文件，相当于给 AI 的入职文档。里面应该写：项目技术栈和约定、常用命令（`pnpm dev`、`pnpm build`）、目录结构说明、哪些文件/操作不能碰、代码风格偏好等。写得越清楚，AI 的回答越准确，越少犯"低级错误"。

**指令要具体，避免歧义**

"帮我优化这个组件"远不如"帮我把这个组件里的 `watch` 改成 `computed`，不要改其他逻辑"。指令越具体，AI 越不容易跑偏，也更容易审查。

**善用 memory 跨会话保存经验**

Claude Code 支持把重要的项目信息、踩过的坑、用户偏好存到 `memory/` 目录，下一个会话可以继续利用，不需要每次重新解释上下文。

**用 MCP 扩展工具能力**

Claude Code 支持 MCP 协议，可以接入数据库、文件系统、GitHub、Slack 等外部工具。把常用操作封装成 MCP 工具，AI 可以直接调用，不需要反复粘贴内容。

**分步骤处理复杂任务**

复杂任务不要一次性丢给 AI，容易跑偏。先让 AI 列出计划，确认方向对了再逐步执行，每一步完成后 review，发现问题及时纠正。

**用 `/compact` 管理上下文**

对话过长时，上下文里堆积了大量中间过程，会干扰 AI 对当前任务的理解。用 `/compact` 压缩历史，让 AI 聚焦当前状态。

### 如何让AI在回答中有效利用上下文

核心思路是**减少 AI 需要猜测的东西**，把隐性知识显式化。

**结构化提供背景**

在问题里把"现在的情况"、"期望的结果"、"已经尝试过什么"说清楚。AI 的上下文窗口有限，提供的信息越精准，它利用上下文的效率越高。

**利用 RAG（检索增强生成）**

当知识库很大时，不能把所有内容都塞进上下文，可以先检索出最相关的片段再拼接到 prompt 里。这是让 AI 精准利用上下文的工程化手段，在构建 AI 应用时非常常见。

**CLAUDE.md / 系统提示词**

持久化的项目约定、规范、术语表等放在系统提示词或 CLAUDE.md 里，每次对话都自动带入，而不是靠用户每次手动说明。

**控制上下文噪音**

对话历史过长时，早期的无关内容反而会干扰 AI 理解当前意图。适时 `/compact` 或开新对话，让上下文更聚焦。

**明确引用**

"帮我修改上面那个函数"不如"帮我修改 `src/utils/format.ts` 里的 `formatDate` 函数"，明确引用比模糊指向更能让 AI 定位到正确的上下文。

### CLAUDE.md的作用

CLAUDE.md 是 Claude Code 的**持久化项目级指令文件**，放在项目根目录（或子目录），Claude 每次启动时会自动读取。

**核心作用：给 AI 做项目入职**

不需要每次对话都重复解释"这个项目用 Vue 3 + TypeScript + pnpm"、"不要用 `var`"、"提交前要跑 `pnpm lint`"，这些写进 CLAUDE.md 后，AI 每次都会遵守。

**典型写法**

```markdown
# 项目说明

技术栈：Vue 3 + TypeScript + Vite + pnpm

## 常用命令

- `pnpm dev` - 启动开发服务器
- `pnpm build` - 生产构建
- `pnpm lint` - 代码检查

## 约定

- 使用 Composition API + `<script setup>`，禁止 Options API
- 组件命名 PascalCase，文件名 kebab-case
- 状态管理统一用 Pinia，禁止直接挂到 window

## 禁止操作

- 不要直接修改 `dist/` 目录
- 不要提交 `.env.local`
- 没有明确要求不要安装新依赖
```

**多层级支持**

CLAUDE.md 可以放在不同目录层级，子目录里的 CLAUDE.md 会在对应范围内生效，可以给特定模块写专属约定。

**和 memory 的区别**

CLAUDE.md 是项目级别的、跟代码一起提交到 Git 的约定，适合团队共享；`memory/` 是个人跨会话记忆，适合保存个人习惯和踩坑经验。

### 常用的MCP有哪些

MCP（Model Context Protocol）是 Anthropic 推出的开放协议，让 AI 以统一标准调用外部工具。常用的 MCP Server 按场景分：

**文件与系统**

- `@modelcontextprotocol/server-filesystem`：读写本地文件，Claude Code 默认集成
- `@modelcontextprotocol/server-memory`：跨会话持久化记忆

**代码与开发**

- `@modelcontextprotocol/server-github`：操作 GitHub 仓库，查看/创建 PR、Issue、提交等
- `@modelcontextprotocol/server-gitlab`：GitLab 版本
- `@modelcontextprotocol/server-git`：本地 Git 操作

**数据库**

- `@modelcontextprotocol/server-postgres`：查询 PostgreSQL
- `@modelcontextprotocol/server-sqlite`：查询本地 SQLite

**浏览器与网页**

- `@modelcontextprotocol/server-puppeteer`：控制无头浏览器，截图、爬取动态页面
- `@modelcontextprotocol/server-fetch`：抓取网页内容

**协作与通知**

- `@modelcontextprotocol/server-slack`：发送 Slack 消息、读取频道
- `mcp-server-linear`：Linear 项目管理，创建/更新任务

**思维与推理**

- `@modelcontextprotocol/server-sequential-thinking`：引导 AI 分步骤推理，适合复杂分析场景

实际项目里最常用的组合是 `filesystem + github + postgres`，可以让 AI 直接读代码、操作 PR、查数据库，大幅减少手动粘贴上下文的成本。
