面试官提问：

## 主包从 1.8MB 降到 1.3MB，具体做了哪几件事？每件事分别贡献了多少？

1、第一是分包优化，把非核心页面迁到分包，同时写了 Webpack + Babel 插件做分包异步化，实现小程序跨分包加载
2、第二是清理冗余代码和废弃的 JSON 数据，主要是历史迭代积累的死代码和不再使用的配置。
3、图片资源上 CDN，把本地静态图片全部替换成 CDN 链接。

## Webpack/Babel 插件实现分包异步化引入的原理是什么？本质上是把 import 改成了什么？

整体分两层，Babel 插件负责编译期改写，Webpack 插件负责构建期分包。

Babel 插件遍历 AST，识别特定标记的 import 语句，将其转换为 **non_webpack_require**.async() 调用，Webpack 编译后就变成了小程序原生支持的 require.async()，实现运行时跨分包异步加载。

Webpack 插件则通过自定义 SplitChunks 的 cacheGroups，把原本会打进主包 vendor 的公共依赖，按使用范围重新分配到对应分包目录中，避免主包体积膨胀。

## "编译期组件自动化注入"具体是怎么做的？是 Vite 插件还是 Webpack loader？

使用的是 Vite 插件 @uni-ku/root，它的原理是在编译阶段通过 Vite 的 transform 钩子拦截每个页面的 .vue 文件，自动在 template 中注入弹窗组件标签，在 script 中注入对应的 import 和注册逻辑。支持按需注入，也可以配置排除特定页面。

之所以需要编译期注入，是因为小程序的架构和 Web 不同，每个页面是独立的渲染层，没有全局 DOM，弹窗组件必须存在于当前页面的 template 中才能渲染，不可能像 Web 端那样动态挂载到 body 上。手动在几百个页面写弹窗模板不现实，所以通过编译期自动注入来消除这个重复工作。

## 运行时 Vue 3 响应式 Map 接管全局状态——Map 的 key 是什么，value 的结构是什么？

Map 的 key 是通过组件名和可选的 selector 拼接生成的唯一标识，selector 的设计是为了支持同一个组件的多实例场景。

value 是一个 ref 对象，主要包括 控制显示隐藏、层级控制、业务配置、确认回调、取消回调

show() 创建 Promise → 把 resolve/reject 包装成 success/fail 闭包存进 options → 弹窗组件点确认调 options.success(res)，内部先 close 再 resolve → 点取消调 options.fail(res)，内部先 close 再 reject。
